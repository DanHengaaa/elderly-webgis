## 第一部分 · 产品总览（喂给 AI 的全局上下文）

**产品名称**：养老服务智慧决策 WebGIS 平台（面向银发经济的养老资源空间适配评估与智慧决策系统）

**一句话定位**：一个集「空间分析 — 智能评估 — 精准推荐 — 社区互动 — 智慧问答」于一体的、以地图为核心的养老择址决策平台，帮助家属为老人挑选最合适的养老机构。

**核心差异点**：不是传统「电子黄页」，而是以 WebGIS 地图为中心，叠加等时圈、医疗可达性、生活/生态图层，并配备一个能把自然语言转成空间查询的「AI 伴诊助手」。

**平台形态**：B/S 架构的 Web 应用（桌面浏览器为主，需兼容平板）。前端技术栈为 Vue/React + OpenLayers/Leaflet（地图）+ ECharts（图表）。

**三类用户角色（务必区分三套界面）**：
- **客户端（家属 / 老人）**：地图浏览、空间分析、智能问卷、推荐、AI 伴诊、评价与社区、个人养老档案。**这是产品主界面，必须做适老化设计。**
- **机构端（养老院）SaaS 工作台**：入驻申请、床位/价格维护、实景图库、查看本机构画像与评价、回复评价。
- **管理端（平台方）后台**：入驻审批、数据治理、UGC 内容风控、四维评估调度、全局运营看板。

**整体气质关键词**：可信赖、温暖、专业、清晰、安心。避免冰冷的纯医疗风，也避免花哨的消费 App 风。介于「政务/医疗的严谨」与「生活服务的亲和」之间。

## 第二部分 · 设计系统规范（Design System）

### 2.1 色彩

| 用途 | 颜色 | HEX |
| --- | --- | --- |
| 主色 Primary（信任 / 健康 / 自然） | 深青绿 | `#2E7D6B` |
| 主色浅 | 浅青绿 | `#E3F1ED` |
| 强调色 Accent（CTA / 行动召唤） | 暖橙 | `#E8845B` |
| 成功 | 绿 | `#3A9E5C` |
| 警告 | 琥珀 | `#E0A030` |
| 危险 / 失能提示 | 红 | `#D9534F` |
| 信息 | 蓝 | `#3A7BD5` |
| 主文字（高对比） | 近黑 | `#1F2933` |
| 次要文字 | 灰 | `#52606D` |
| 边框 / 分割线 | 浅灰 | `#D9E0E8` |
| 页面背景 | 极浅灰 | `#F5F7F9` |
| 卡片 / 表面 | 白 | `#FFFFFF` |

**地图等时圈 / 可达性分级专用色**（用于地图着色、徽标、图例，全平台统一）：
- 优（≤15 分钟 / 高保障）：`#2E9E5B` 绿
- 良（15–30 分钟）：`#A8C84A` 黄绿
- 中（30–60 分钟 / 中保障）：`#E8A030` 橙
- 弱（>60 分钟 / 低保障）：`#D9534F` 红

### 2.2 字体与排版（适老化：整体偏大）

- **字体**：中文用「思源黑体 / 苹方 / Microsoft YaHei」，数字与英文用「Inter」。
- **正文基础字号 18px**（比常规 Web 偏大，适老化）；老年模式可一键放大到 20–22px。
- 标题：H1 = 32px / Bold，H2 = 26px / Semibold，H3 = 22px / Semibold，H4 = 18px / Medium。
- 行高宽松，正文 1.6；段落间距充足。
- 关键数字（评分、床位、月费、耗时）用大号加粗强调。

### 2.3 形状、间距与组件基线

- 圆角：卡片 12px，按钮 8px，输入框 8px，标签/徽标 999px（胶囊）。
- 间距系统：8 的倍数（8 / 16 / 24 / 32 / 40）。
- 阴影：轻柔（`0 2px 8px rgba(0,0,0,0.06)`），不要重投影。
- **触控目标 ≥ 48px 高**（适老化），主按钮高 52–56px。
- 图标统一线性风格（如 Lucide / Tabler 风格），**关键操作图标必须配文字标签**，不要只放图标。

### 2.4 适老化设计原则（客户端必须遵守）

1. 大字体、大按钮、宽间距、高对比；正文与背景对比度 ≥ 7:1。
2. 关键入口（地图、问卷、AI 助手、推荐）必须在首屏显眼可见，层级浅。
3. 提供**语音输入**入口（AI 助手、搜索框旁放麦克风图标按钮）。
4. 状态/反馈明确：加载、空状态、成功、错误都要有清晰提示。
5. 重要不可逆操作（提交问卷、确认对比、删除）需二次确认。
6. 提供「老年模式」开关：放大字体、提高对比、简化页面。

## 第三部分 · 全局布局与导航

### 客户端布局
- **顶部导航栏**（固定）：左侧 Logo + 平台名；中间主导航「地图找院 / 智能评估 / 我的推荐 / AI 伴诊 / 社区」；右侧「老年模式开关、消息、用户头像下拉（个人养老档案 / 收藏 / 退出）」。
- 大量页面采用「**左侧地图 + 右侧面板**」或「**左侧筛选 + 右侧列表/地图**」的双栏布局。
- 移动/平板下导航折叠为底部 Tab。

### 机构端 SaaS 布局
- **左侧竖向侧边栏导航**（工作台总览 / 机构资料 / 床位价格 / 实景图库 / 评价管理 / 入驻状态）+ 顶部面包屑与机构名。
- 内容区为表单与数据卡片，偏 SaaS 后台风格但仍温暖清晰。

### 管理端布局
- **左侧深色侧边栏 + 顶部栏**的经典后台布局。导航：运营看板 / 入驻审批 / 数据治理 / 内容风控 / 四维评估。
- 大量表格、状态标签、筛选器、地图诊断视图。

## 第四部分 · 逐页面详细规格

> 下列每个页面块都可单独复制给 Figma AI 生成。每块包含：**目的 / 布局 / 核心元素与内容 / 交互状态**。内容请用文档中给出的真实示例数据，不要用 Lorem Ipsum。

### 【A · 客户端】

#### A1. 登录 / 注册页
- **目的**：客户多端登录入口。
- **布局**：左右分栏。左侧为品牌区（平台名 + 一句话定位 + 一张温暖的养老/地图主题插画或地图截图背景）；右侧为登录卡片。
- **核心元素**：手机号输入框 + 「获取验证码」按钮、验证码输入框；下方「微信 / 支付宝快捷登录」两个第三方按钮（圆形图标）；勾选《用户协议》；大号主按钮「登录 / 注册」。底部小字「机构端入口」「管理端入口」链接。
- **状态**：验证码倒计时（60s）、错误提示（手机号格式 / 验证码错误）。

#### A2. 地图主页（核心 WebGIS 工作台）★ 最重要
- **目的**：以地图为核心，展示养老机构及周边空间环境，承载等时圈、图层叠加、可达性分析。
- **布局**：**全屏地图为底**，左侧为可收起的「筛选 + 图层控制」面板，右侧为「机构列表 / 详情」抽屉，顶部为搜索与起点设置条。
- **核心元素**：
  - 顶部：地址搜索框（带麦克风语音输入）、「设置居住地/通勤起点」输入、出行方式切换（驾车/公交/步行三个图标按钮）、等时圈档位切换（15 / 30 / 60 分钟 三个胶囊按钮）。
  - 地图区：底图 + 养老机构点位（按综合评分或可达性分级着色）+ 半透明等时圈多边形（按 2.1 分级色着色）+ 用户起点标记。地图右下角有缩放、定位、图例。
  - 左侧**图层控制器**（可勾选叠加）：医疗设施（三甲/综合/卫生院/专科分组）、生活服务（菜市场/超市/药店/银行/社区食堂）、公园景点、负面 POI；每类带颜色图例。
  - 左侧**筛选**：机构性质（公办公营/公办民营/民营）、等级、价位档（经济/标准/高端）、空余床位、护理类型（自理/半失能/失能/失智）。
  - 右侧**机构列表卡片**：每张卡显示 机构名称、性质标签、等级、距离/探视耗时（带分级色徽标）、综合评分（星级）、起价月费、空余床位、一张缩略实景图、「查看详情」「加入对比」按钮。
- **交互**：拖动起点实时重算等时圈并筛选落在圈内的机构；点击机构点高亮并弹出右侧详情；图层勾选即时叠加。

#### A3. ADL 智能评估问卷页
- **目的**：把国际通用 ADL（日常生活活动能力）量表做成交互式问卷，逐项作答生成护理等级。
- **布局**：居中单列卡片式问卷，顶部步骤进度条（如「身体自理 → 经济预算 → 位置偏好 → 生活休闲 → 完成」）。
- **核心元素**：
  - 每题一屏或一组：题干 + 大号单选选项卡（如「进食：完全自理 / 需部分协助 / 完全依赖」）。
  - 经济维度：预算区间滑块或档位选择（经济型 / 标准型 / 高端型）。
  - 位置维度：居住地 + 期望择址范围（可达性约束）。
  - 生活休闲维度：多选标签（医疗优先 / 环境优先 / 餐饮 / 娱乐）。
  - 底部「上一步 / 下一步」大按钮；最后一步显示「生成我的养老画像」。
- **完成页**：展示生成的护理等级标签（自理 / 半失能 / 失能 / 失智 等，带说明），以及「查看为我推荐的机构」主按钮。
- **适老化**：选项卡片足够大、文字清楚，可配小图标辅助理解。

#### A4. 智能推荐列表页
- **目的**：按综合匹配度降序展示推荐机构。
- **布局**：顶部「我的画像摘要条」（护理等级 + 预算 + 区域，可点「重新评估」）+ 排序/筛选条；下方为推荐卡片列表（可切换列表/地图视图）。
- **核心元素**：每张推荐卡：排名序号、机构名称、综合匹配度得分（大号，0–100 或星级）、**各分项指数雷达缩略图**（探视可达 / 医疗可达 / 急救响应 / 生活便利 / 绿色休闲）、关键标签（公办、设医务室、无障碍…）、起价月费、空余床位、「查看详情」「加入对比」「连访路线」按钮。
- **交互**：可调整各维度权重（弹出滑块面板，对应 AHP 权重），列表实时重排。

#### A5. 机构详情页 ★
- **目的**：单机构全方位画像，支撑线上初筛。
- **布局**：顶部实景图轮播/相册（支持全景看房入口）+ 机构名称与核心标签；下方分区：基本信息、空间画像、口碑评价、地图位置、服务与设施。
- **核心元素**：
  - **基本信息卡**：性质（公办公营/公办民营/民营）、等级、地址、联系电话、总床位 / 空余床位、起价月费。
  - **双雷达图并排**：左「客观空间指数雷达图」（探视可达/医疗可达/急救响应/生活便利/绿色休闲，含综合空间宜居指数）；右「主观口碑雷达图」（医疗响应/硬件设施/餐饮膳食/护理服务/性价比）。
  - **医疗生命线小卡**：周边三甲/综合医院数量与最近距离、急救响应能力徽标（高/中/低保障，分级色）。
  - **服务能力**：照护服务标签（自理/半失能/失能/失智/认知症照护/术后康复）。
  - **设施设备**：电梯 / 康复理疗室 / 阅览室 / 独立花园 / 无障碍通道 等图标网格。
  - **位置地图**：嵌入小地图，显示机构点 + 等时圈 + 可叠加周边图层。
  - **评价区**：综合评分 + 评价列表（用户、评分、文字、图片、时间），「写评价」按钮。
  - 右侧/底部固定操作条：「收藏」「加入对比」「问 AI 伴诊」「预约 / 联系」。

#### A6. 机构对比页
- **目的**：多家机构横向对比。
- **布局**：横向对比表格，列为机构（最多 4 个），行为对比维度。
- **核心元素**：每列顶部机构缩略图 + 名称 + 移除按钮；对比行包括：综合匹配度、性质、等级、月费、空余床位、探视耗时、医疗可达、急救响应、生活便利、绿色休闲、综合评分、护理类型。差异行可高亮最优值。底部「规划连访路线」按钮。

#### A7. AI 伴诊助手页 ★ 技术亮点
- **目的**：自然语言对话即可解析需求、调度 GIS 空间查询与匹配，并联动地图。
- **布局**：**左侧对话区 + 右侧地图/结果区**的双栏。
- **核心元素**：
  - 左侧聊天：消息气泡（用户 / AI）、底部输入框带**麦克风语音按钮**和发送按钮；AI 回复支持流式（打字机）效果；常见问题快捷气泡（如「帮我找鼓楼区 30 分钟车程、有医务室、月费 5000 以内的养老院」）。
  - AI 回复中嵌入**可联动地图的机构卡片**：点击卡片即在右侧地图高亮对应机构、绘制其等时圈与周边图层。
  - 右侧地图：随对话动态变化，显示被推荐机构、等时圈、医疗点等。
- **状态**：AI 思考中（loading）、澄清追问（多轮对话，如「您更看重离家近还是医疗强？」）、空状态引导语。

#### A8. 连访路线规划页
- **目的**：为收藏/对比的多家机构生成一次出行可依次到访的最优路线。
- **布局**：左侧地图显示路线 + 顺序编号点位；右侧为按顺序排列的机构站点列表（可拖动调整顺序）+ 出行方式切换 + 总耗时/总里程。
- **核心元素**：站点卡（序号、机构名、预计到达时间、停留建议）、「导出/分享路线」按钮。

#### A9. 个人养老档案 / 个人中心
- **目的**：「一人一档」结构化画像，支撑匹配与 AI。
- **布局**：左侧菜单（家庭与居住信息 / 老人个人情况 / 偏好与预算 / 收藏与历史 / AI 对话历史）+ 右侧表单/列表。
- **核心元素**：
  - 家庭与居住：常住地址（自动地理编码显示在小地图上）、子女通勤起点、期望择址范围。
  - 老人情况：年龄、基础疾病标签、ADL 自理等级（来自问卷）、护理偏好。
  - 偏好与预算：价位偏好、医疗配套要求、休闲娱乐偏好。
  - 收藏与历史：收藏机构卡片网格、浏览历史、AI 对话历史、已保存的连访路线。

#### A10. 内容社区页（UGC 论坛 + 护理科普）
- **目的**：真实家属口碑与护理科普分享。
- **布局**：顶部分类 Tab（择址经验 / 护理科普 / 问答 / 政策补贴）+ 搜索框；中间帖子信息流；右侧「热门话题 / 推荐机构」侧栏。
- **核心元素**：帖子卡（作者头像、标题、摘要、图片、点赞/评论数、标签、时间）；「发帖」浮动按钮；全文检索框。

#### A11. 发表评价页
- **目的**：提交多维评价（雷达图数据来源）。
- **布局**：居中表单卡片。
- **核心元素**：选择机构、综合评分（5 星）、**四个维度分项打分**（医疗响应 / 硬件设施 / 餐饮膳食 / 护理服务，各 5 星）、文字内容、图片上传（多图）、提交按钮。提交后提示「将经管理员审核后展示」。

### 【B · 机构端 SaaS 工作台】

#### B1. 工作台总览（Dashboard）
- **目的**：机构运营概览。
- **布局**：左侧侧边栏 + 右侧卡片网格。
- **核心元素**：顶部关键指标卡（被收藏次数、到访统计、空余床位、综合评分）；本机构**多维评价雷达图**；待回复评价提醒；入驻/资质状态徽标。

#### B2. 机构入驻申请页
- **目的**：标准化入驻通道。
- **布局**：分步表单（机构基本信息 → 资质上传 → 提交审核）+ 顶部步骤条。
- **核心元素**：机构名称/地址/联系人/电话表单；资质上传区（营业执照、养老机构备案证明，拖拽上传）；**状态机进度提示**（待审核 → 审核中 → 通过 / 驳回，驳回显示原因）。

#### B3. 床位与价格维护页
- **目的**：动态维护床位与价格，实时同步前端匹配。
- **核心元素**：总床位 / 空余床位编辑（数字步进器）、护理等级分区表格（自理/半失能/失能/失智 各自床位与价格档位）、月费档位编辑、「保存并同步」按钮、最近更新时间。

#### B4. 实景图库 / 房型管理页
- **目的**：上传实景图、房型图与全景素材。
- **布局**：图片网格管理界面。
- **核心元素**：分组（机构实景 / 房型图 / 全景）、拖拽上传、封面设置、排序、删除；全景素材挂载入口。

#### B5. 评价管理与回复页
- **目的**：查看并回复本机构评价。
- **布局**：评价列表 + 回复抽屉。
- **核心元素**：评价卡（用户、各维度评分、内容、图片、时间、审核状态）、「回复」输入框、回复历史。

### 【C · 平台管理端】

#### C1. 运营数据看板
- **目的**：全局运营驾驶舱。
- **布局**：深色侧边栏 + 顶部栏 + 卡片/图表网格。
- **核心元素**：核心指标卡（机构总数、客户数、UGC 内容数、待审批数）；机构区域分布图（地图）；入驻趋势折线图；各区机构数/床位数柱状图；待办（待审批、待风控）列表。

#### C2. 机构入驻审批页（状态机）
- **目的**：审批机构入驻与资质变更。
- **布局**：左侧状态筛选 Tab（待审核 / 审核中 / 通过 / 驳回）+ 右侧申请列表表格。
- **核心元素**：表格列（机构名、区县、提交时间、资质材料预览、状态标签）；详情抽屉（查看营业执照/备案证明、「通过 / 驳回」按钮、驳回原因填写、审批日志）。

#### C3. POI / 机构数据治理页
- **目的**：对爬取的 POI、机构数据去重、纠偏、地理编码质检。
- **布局**：左侧数据集切换（医疗 / 生活 / 公园 / 机构）+ 右侧数据表 + 右侧地图校验视图。
- **核心元素**：数据表（名称、类别、经纬度、区县、坐标系标记、状态）；疑似重复/坐标偏移高亮；在地图上核对点位并手动纠偏；批量地理编码质检入口。

#### C4. UGC 内容风控审核页
- **目的**：敏感词过滤与人工复核。
- **布局**：左侧队列（待审 / 已通过 / 已拦截）+ 右侧内容详情。
- **核心元素**：内容卡（评价/帖子内容、图片、命中的敏感词高亮、风险等级）、「通过 / 拦截 / 退回」操作、审核记录。

#### C5. 四维评估与盲区诊断页（地图）
- **目的**：将「空间可达性 / 分配均等性 / 供需适配性 / 社会正义性」四维测度结果可视化，辅助政府规划与养老盲区诊断。
- **布局**：**左侧大地图（街道单元面着色）+ 右侧指标面板**。
- **核心元素**：
  - 地图：以街道单元（town_unit）为面，按所选维度指数做分级设色（choropleth），可切换四个维度；盲区（高需求低供给）用红色高亮。
  - 右侧：维度切换、阈值/分级图例、选中单元的详情卡（老年人口、床位数、供需比、各维度得分）、Top 盲区列表、导出报告按钮。

## 第五部分 · 关键可复用组件库

请 AI 把以下组件做成统一可复用样式：

1. **机构卡片（Institution Card）**：实景缩略图 + 名称 + 性质/等级标签 + 可达性分级徽标 + 综合评分星级 + 月费 + 空余床位 + 操作按钮。三种尺寸（地图侧栏小卡 / 推荐大卡 / 对比列）。
2. **可达性 / 保障分级徽标（Pill Badge）**：用 2.1 分级色，文字「优/良/中/弱」或「高/中/低保障」。
3. **雷达图卡（Radar Chart）**：5 维，用于空间指数与口碑评价两套，配色一致。
4. **状态标签（Status Tag）**：待审核(灰)/审核中(蓝)/通过(绿)/驳回(红)/已上架/已下架。
5. **图层控制器（Layer Control）**：分组可勾选 + 颜色图例。
6. **等时圈档位切换器**：15/30/60 分钟胶囊按钮 + 出行方式图标切换。
7. **语音输入按钮**：圆形麦克风按钮，录音态有波纹动画。
8. **空状态 / 加载 / 错误**：统一的占位插画与提示文案。
9. **星级评分组件**：可读 + 可交互两种。

## 第六部分 · 内容/数据字段参考（让生成的页面填充真实数据，而非占位符）

- **机构示例字段**（来自数据库 institutions 表）：机构名称、机构性质（1 公办公营 / 2 公办民营 / 3 民营）、等级（五级/四级/三级）、区县（如鼓楼区、栖霞区、江宁区）、总床位、空余床位、基础月费、联系人、联系电话、审核状态（待审核/已上架/已下架）。
- **照护服务**（institution_services）：自理 / 半失能 / 失能 / 失智 / 认知症照护 / 术后康复。
- **设施设备**（institution_facilities）：电梯 / 康复理疗室 / 阅览室 / 独立花园 / 无障碍通道。
- **评价维度**（institution_reviews）：综合评分、医疗响应、硬件设施、餐饮膳食、护理服务（各 1–5 分）。
- **空间指数**（来自 M2）：探视可达性指数、医疗资源可达性指数、急救响应能力、生活便利度指数、绿色休闲指数、综合空间宜居指数（均 0–1，详情页用百分比或评分展示）。
- **POI 类别**（poi 表 category）：1 医疗设施 / 2 生活服务 / 3 公园景点 / 4 负面 POI。医疗小类示例：三级甲等医院、综合医院、卫生院、口腔医院、眼科医院等。
- **区县示例**：南京市下辖 鼓楼区、秦淮区、玄武区、栖霞区、江宁区、高淳区 等。

## 第七部分 · 如何把本文档用于 Figma AI

1. **建立设计系统**：先把「第一 + 第二 + 第三部分」一次性粘贴给 Figma Make / First Draft，提示它：「这是一个 Web 平台的设计系统与全局规范，请据此建立颜色、字体、组件库样式。」
2. **逐页生成**：从「第四部分」复制单个页面块（如 A2 地图主页），提示：「按以下规格生成一个桌面端 Web 页面，使用上面建立的设计系统，内容用文中真实示例数据，不要用 Lorem Ipsum。」
3. **强调约束**：在每次提示末尾追加一句固定约束，例如：
   > 「平台为中文、桌面 Web、适老化（大字体大按钮高对比）；地图相关页面请用占位地图图像表示，机构点、等时圈、图层按规格着色；图表用雷达图/柱状图占位。」
4. **迭代**：先生成低保真，再让 AI 逐页加细节（轮播、雷达图、抽屉、状态标签）。地图与图表 Figma AI 通常用图片/矢量占位即可，真实交互在前端框架里实现。
5. **优先级建议**：先做客户端 A2（地图主页）、A5（机构详情）、A7（AI 伴诊）、A4（推荐列表）这四张核心页，它们最能体现产品价值；机构端与管理端可后续批量生成。
